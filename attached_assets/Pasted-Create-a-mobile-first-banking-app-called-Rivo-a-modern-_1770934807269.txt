DATABASE SCHEMA:

Users table: id, full_name, email, password_hash, phone, created_at, updated_at
Cards table: id, user_id, card_number, card_holder, expiry_date, cvv, is_frozen, spending_limit, created_at
Transactions table: id, user_id, card_id, type (debit/credit), amount, category, merchant, description, date, status
Rewards table: id, user_id, points, cashback_amount, source, date
Beneficiaries table: id, user_id, name, account_number, bank_name, avatar
Notifications table: id, user_id, title, message, type, is_read, created_at

API ENDPOINTS TO CREATE:

AUTH:
POST /api/auth/signup - Create new user
POST /api/auth/login - Login user
POST /api/auth/logout - Logout user
GET /api/auth/me - Get current user

CARDS:
GET /api/cards - Get user's cards
POST /api/cards - Create new virtual card
PUT /api/cards/:id/freeze - Freeze/unfreeze card
PUT /api/cards/:id/limit - Update spending limit

TRANSACTIONS:
GET /api/transactions - Get all transactions (with filters)
POST /api/transactions - Create new transaction
GET /api/transactions/:id - Get single transaction
DELETE /api/transactions/:id - Delete transaction
GET /api/transactions/stats - Get spending analytics

TRANSFERS:
POST /api/transfers/send - Send money
POST /api/transfers/request - Request money

REWARDS:
GET /api/rewards - Get user rewards
POST /api/rewards/redeem - Redeem points

BENEFICIARIES:
GET /api/beneficiaries - Get saved beneficiaries
POST /api/beneficiaries - Add beneficiary

NOTIFICATIONS:
GET /api/notifications - Get all notifications
PUT /api/notifications/:id/read - Mark as read
